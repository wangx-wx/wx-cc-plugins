# SQL/XML 检查清单
> 规则编号采用 `SQL-00001` 形式递增，级别枚举：`Critical` > `Major` > `Minor` > `Info`
> 规则标题格式：`规则编号 规则名称`

## SQL-00001 禁止使用 SELECT *
- 级别：Critical
- 描述：Mapper XML 中存在 `SELECT *` 或 `select *`。当表结构变更时会导致字段映射异常、返回冗余数据，增加网络传输和内存开销
- 修复建议：将 `*` 替换为实际需要的字段列表；若字段较多可抽取 `<sql id="Base_Column_List">` 片段复用

## SQL-00002 DML 缺失 WHERE 条件
- 级别：Critical
- 描述：UPDATE 或 DELETE 语句中完全没有 WHERE 关键字，执行时将作用于全表，可能导致数据批量误更新或误删除
- 修复建议：补充 WHERE 条件限定操作范围，至少包含主键或唯一索引字段

## SQL-00003 DML 条件完全被 `<where>` 标签包裹导致条件失效
- 级别：Critical
- 描述：UPDATE 或 DELETE 语句的所有 WHERE 条件均位于 `<where>` 标签内的 `<if>` 中。当所有 `<if>` 条件均不满足时，`<where>` 标签不生成任何内容，最终执行无 WHERE 子句的全表操作
- 修复建议：至少将一个必要条件（如主键）硬编码为固定 WHERE 子句，仅将可选过滤条件放入 `<if>` 标签

## SQL-00004 SQL 注入风险（`${}` 拼接）
- 级别：Critical
- 描述：MyBatis 中 `${}` 将参数值直接拼接进 SQL 字符串，不做转义。当参数来源于用户输入时，攻击者可注入任意 SQL 片段，导致数据泄露或篡改
- 修复建议：所有用户可控参数必须使用 `#{}` 预编译占位符；仅在动态表名、列名等无法参数化的场景使用 `${}`，且必须配合白名单校验
